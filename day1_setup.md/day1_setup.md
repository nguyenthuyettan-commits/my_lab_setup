Day 1: Setup & Lab Environment – Tool Selection Mindset 
1. Cursor IDE Why it was selected:
   Deep AI Integration (AI-Native): Unlike traditional IDEs that rely on add-on plugins, Cursor embeds AI directly into the core architecture. It understands the full project context, enabling more accurate code suggestions and intelligent debugging support powered by Large Language Models (LLMs).
   Composer & Multi-file Editing: The ability to modify multiple files simultaneously through natural language instructions significantly improves development efficiency and accelerates system implementation.
   Platform Compatibility: Built on top of VS Code, Cursor preserves the familiar developer experience while maintaining compatibility with the entire existing extension ecosystem.
2. Warp Terminal Why it was selected:
   AI-Native Terminal: Natural language command execution directly within the terminal reduces dependency on external searches such as Google or Stack Overflow, improving operational productivity.
   Modern Userxperience: Features such as mouse-based cursor navigation, text selection, and block-based output organization provide a far more intuitive experience when working with logs, scripts, and command outputs compared to traditional terminals.
   Warp Drive & Workflows: Supports storing and sharing reusable commands and workflows, helping standardize operational processes across environments, including local machines and remote servers.
3. Engineering Mindset Tool selection should not be driven solely by popularity, but by performance, scalability, and workflow optimization capabilities.
4. Objective: Build a working environment where AI acts as an engineering copilot, automating repetitive tasks so engineers can focus on system architecture, problem-solving, and core business logic.
